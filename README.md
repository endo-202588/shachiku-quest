## アプリタイトル

社畜クエスト　〜デスクの上の大冒険〜

## サービス概要

**社畜クエスト**は、チーム内の業務量を可視化し、助け合いを促進する業務管理アプリです。

### 解決したい課題

- 誰がどのくらい忙しいのか分からない
- 助けたいけど、誰が助けを求めているか分からない
- 業務の偏りが見えにくい
- 管理職が社員の状況を把握しづらい

### 提供する価値

- その日の業務量を「ステータスコード」で簡単に共有
- 助けられる人と助けが必要な人をマッチング
- RPGゲームの世界観で楽しく協力を促進
- 管理職は社員の状況を一目で把握できる

---

## このサービスへの思い・作りたい理由

普段誰がどのくらいの業務負担があるのか、なかなか把握できないことをもどかしく思っていました。

そこで、"猫の手も借りたい"人と、"助けられます"という人を楽しくつなげて業務量の平均化を図り、助けた人も経験値を積むことでレベルアップしていく様子を楽しめるアプリを作りたいと考えました。

実用的な面では、その日のチームメイトの業務量が可視化され、お休みも含めて社員の状況が把握できる、管理職にとっては簡単に社員の負担度がつかめる、というメリットがあります。

## サービスの利用イメージ

### 1. 朝の業務開始時

各社員（勇者）は、出勤したらその日の業務量を登録します。

**ステータスコード（業務量）**

- 🟢 **平和** - 余裕があります（ヘルプ不要）
- 🟡 **疲れ気味（0.5h）** - ちょっと忙しい
- 🟠 **やばいかも（1h）** - かなり忙しい
- 🔴 **きてます（1.5h）** - 助けてください！
- ⚫ **昇天（2h〜）** - 限界です！
- ○ **お休み** - お休み

### 2. シャチークの酒場（ヘルプマッチング）

助けられる人は「使える魔法（お手伝い可能時間）」を登録します。

**使える魔法**

- ⚡ **ベハーフ（0.5h）** - ちょっとだけお手伝い
- ✨ **ベホワン（1h）** - しっかりお手伝い
- 💫 **ベホワンハフ（1.5h）** - がっつりお手伝い
- 🌟 **ベホロング（2h〜）** - 全力でお手伝い

### 3. ヘルプ依頼（シャチークの酒場）

「シャチークの酒場」は、助けが必要な人と助けられる人をマッチングする場所です。

- 現在ヘルプが必要なタスクの表示
- ヘルプができる勇者の名前・使える魔法の表示

#### ヘルプ依頼の流れ

1. **ユーザーA（助けが必要な人）**

   - タスク「資料作成」をヘルプ依頼に登録（所要時間：1時間）
   - 「シャチークの酒場」で自分のタスクに合った魔法（時間）の勇者を選び、ヘルプ依頼を出す

2. **ユーザーB（助けられる人）**

   - 「使える魔法（お手伝い可能時間）」を登録
   - 「シャチークの酒場」でヘルプ依頼一覧を確認
   - ヘルプを依頼されるのを待つもしくは自分の「使える魔法（お手伝い可能時間）」に合ったタスクを選択
   - 依頼を受けるか、自分から「手伝う」ボタンを押して受諾

3. **タスク完了**
   - ユーザーBがタスクを完了
   - ユーザーAが完了確認
   - ユーザーBが「徳（ポイント）」を獲得し、レベルアップに繋がる

#### 補足

- 助けられる人は、特定のタスクを自分からピンポイントで助けることも可能
- 完了すると「徳（ポイント）」を獲得し、レベルアップに繋がる

### 4. レベルアップ

一定の徳を得ると、レベルがアップします！

**レベル一覧**

- 🌱 **旅人** - 初心者（0〜99徳）
- 🌿 **先輩** - 経験者（100〜299徳）
- 🌳 **先生** - ベテラン（300〜599徳）
- 🏆 **達人** - エキスパート（600〜999徳）
- 🙏 **仏** - マスター（1000徳〜）

## サービスの差別化ポイント・推しポイント

業務管理アプリのようなものは多く存在していますが、このように"助けを求める、助ける"といったアプリはあまり見られません。
しかもそれをRPGゲームの代表作"ドラゴンクエスト"のパロディのような形でゲーム仕様にし、社員が楽しみながら協力することを促進するという発想は非常にユニークであり、実用的でありながらもエンタメ性のあるアプリになることが期待できると思います。

## 機能候補

### MVP（最小限の機能）

- [ ] ユーザー登録・ログイン機能（Sorcery）
- [ ] 勇者（社員）のプロフィール登録

#### ステータス管理

- [ ] ステータスコード登録（業務量の入力）
- [ ] 本日の勇者一覧（チームメンバーの状況確認）

#### タスク管理

- [ ] タスク登録（タイトル・説明・所要時間・期限）
- [ ] タスク一覧表示

#### ヘルプ依頼機能（核心機能）

- [ ] シャチークの酒場（ヘルプ依頼一覧の表示）
- [ ] 使える魔法の登録（お手伝い可能時間の登録）
- [ ] ヘルプ依頼の作成・受諾・完了
- [ ] 徳（ポイント）の付与
- [ ] レベルアップ機能

### 追加機能（フェーズ2）

- [ ] 通知機能（ヘルプ要請のマッチング成立時、ヘルプ要請完了時、ヘルプ要請クローズ時）
- [ ] チャット機能（ヘルプ要請のマッチング成立時、当該ヘルプ要請タスク内で、チャット形式でメッセージのやりとり）
- [ ] ランキング機能（徳ポイント順）
- [ ] 統計・分析機能（業務量の推移）

### 将来的な機能（フェーズ3）

- [ ] チーム・部署の管理
- [ ] バッジ・称号システム

## 使用する技術スタック

Rails初心者ということもあり、まずはカリキュラムで学んだ技術でできることを中心に、復習の意味を込めて作りたいと思います。

### フレームワーク

- Ruby on Rails 7.1
- Tailwind CSS

### データベース

- PostgreSQL

### デプロイ先

- Render

### 使用予定のライブラリ

**認証・基本機能**

- `sorcery` - 認証機能
- `kaminari` - ページネーション
- `rails-i18n` - 日本語化
- `dotenv-rails` - 環境変数管理

**画像・検索機能**

- `Active Storage` + `image_processing` - 画像アップロード
- `ransack` - 検索機能

**ランキング機能（フェーズ2以降）**

- `redis` + `redis-rails` - ランキング機能
